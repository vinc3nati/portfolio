innerText vs innerHTML

September, 2021

Introduction

When you're starting with javascript understading innerText and innerHTML may seem a little bit overwhelming if you are a complete beginner. No worries, here's a guide which would give a concise idea about how these things works.

More about it

Suppose we have a variable which is an object of a certain output div element in .html file. And we set its innerText to some text with html tags as we want them to render beautifully.( refer below code )

JS

var_name.innerText = "<h1> Hello world!! </h1> <strong> Vinit <strong/> Here"
So here's what we get as an output.

So as we can see the text is rendered as it is without parsing html elements. This is the default behaviour of innerText. i.e. It considers all the content within it as a plain text (string). So no styling is applied.

Now if we replace innerText with innerHTML then we could see the magic. Let's try it out.

JS

var_name.innerHTML = "<h1> Hello world!! </h1> <strong> Vinit </strong> Here"
And we get the following result.
So as we can see innerHTML parses the content as an html document, hence we get the styled output which seemed impossible with innerText So depending on what situations we are dealing with we can leverage both of them in our favour.
